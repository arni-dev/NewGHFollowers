## 注释风格说明

### 1. 文件级注释
每个文件顶部保留原有的版权信息和创建日期。

### 2. 类/结构体/枚举注释
使用三斜线注释（`///`）说明类型的用途和功能：
```swift
/// NetworkManager 类：网络管理器，负责处理所有网络请求和图片下载
/// 使用单例模式确保全局只有一个实例
class NetworkManager {
    // ...
}
```

### 3. 属性注释
为重要属性添加说明：
```swift
/// 单例实例，全局共享
static let shared = NetworkManager()

/// 图片缓存，用于存储已下载的头像图片，避免重复下载
/// NSCache 会在内存不足时自动清理缓存
let cache = NSCache<NSString, UIImage>()
```

使用标准的 Swift 文档注释格式：
```swift
/// 异步下载图片

/// - Returns: 下载的 UIImage 对象，如果下载失败则返回 nil

    // ...
}
```

### 5. 代码内注释
在关键逻辑处添加行内注释，解释代码的意图：
```swift
// 设置 JSON 键名解码策略：将 snake_case 转换为 camelCase
```

## 注释原则

1. **详细但不冗余**：注释应该解释"为什么"和"做什么"，而不是重复代码本身
2. **中文表达清晰**：使用简洁明了的中文，避免生硬的翻译
3. **适合小白**：假设读者是 Swift 初学者，解释关键概念
4. **保持一致性**：所有文件使用相同的注释风格和格式
5. **代码可读性优先**：注释应该提升代码可读性，而不是干扰阅读

## 技术要点说明

### MVVM-C 架构
- **Model**：数据模型（Follower、User）
- **View**：视图控制器（SearchVC、FollowerListVC 等）
- **ViewModel**：视图模型，处理业务逻辑和数据转换
- **Coordinator**：协调器，管理导航流程

### 关键技术
- **Combine**：响应式编程框架，用于数据绑定
- **async/await**：异步编程，用于网络请求和图片下载
- **UserDefaults**：本地数据持久化
- **NSCache**：图片缓存机制
- **Auto Layout**：界面布局
